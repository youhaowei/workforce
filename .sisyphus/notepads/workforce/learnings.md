# Learnings - Workforce

Last updated: 2026-02-09

## What Worked

1. Event-driven service boundaries are effective for streaming UI updates.
2. Explicit lifecycle states reduce orchestration ambiguity.
3. Isolated worktrees are the safest baseline for parallel code agents.
4. Compact planning notes are easier to maintain than long task transcripts.

## Process Learnings

1. Keep `.sisyphus` operational and concise; put canonical product detail in `docs/PRD-MVP.md`.
2. Avoid duplicating milestone details across plan and notepads.
3. Mark resolved problems by removing them from active issue lists instead of accumulating stale history.
